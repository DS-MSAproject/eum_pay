Search Server API 명세 (Frontend)

대상: /api/v1/search/** · 기준일: 2026-04-15

1. 공통 응답 포맷

{
  "status": "success",
  "totalElements": 100,
  "totalPages": 9,
  "currentPage": 0,
  "size": 12,
  "isFirst": true,
  "isLast": false,
  "hasNext": true,
  "hasPrevious": false,
  "extra": {},
  "data": []
}

인증은 게이트웨이 정책에 따르며, 보호 경로에서는 Authorization 헤더가 필요할 수 있습니다.

2. 상품 검색

GET /api/v1/search/products

Query설명
title상품명 검색어
keyword보조 검색어
category대카테고리 (기본: ALL)
subCategory소카테고리
sortType최신순 / 판매량순 / 가격 높은순 / 가격 낮은순
page페이지 (기본 0)

data[] 필드: id, imageUrl, productTitle, originalPrice, price, discountRate, discountTag, isNew, productTag, productUrl, category

3. 베스트셀러

GET /api/v1/search/products/bestseller

현재 구현은 검색 랭킹 기반입니다(실판매량 집계 아님).

data[] 필드: id, imageUrl, productTitle, price, salesRank, rankTag, productUrl

4. 홈 베스트셀러

GET /api/v1/search/products/home-bestseller

Query설명
size노출 개수 (기본 3)

data[] 필드: rank, id, imageUrl, productTitle, price, score, salesCount, createdAt, productUrl

5. 유사 상품 추천 (상세 페이지용)

GET /api/v1/search/products/{productId}/similar

Path/Query설명
productId기준 상품 ID
size반환 개수 (기본 3)

data[] 필드 (슬림 DTO): productId, imageUrl, title, tags, price

tags 규칙: 다중 태그 배열, 우선순위 [NEW] 먼저 후 [판매 1위/2위/3위]

{
  "productId": 185,
  "imageUrl": "https://...",
  "title": "오독오독 사료",
  "tags": ["[NEW]", "[판매 2위]"],
  "price": 12900
}

6. 자동완성

GET /api/v1/search/products/autocomplete

Query설명
name자동완성 입력 문자열

응답은 배열이며 각 항목은 id, title 입니다.

7. 인기 검색어

GET /api/v1/search/products/trending

응답 배열 각 항목: rank, keyword, score

8. 리뷰 검색

GET /api/v1/search/reviews

Query설명
productId상품 ID 필터
keyword리뷰 내용/작성자 검색어
sortTypeBEST 또는 최신순
reviewTypeALL / PHOTO / IMAGE / VIDEO / TEXT
page, size페이징 (기본 0, 5)

9. 공지 검색

GET /api/v1/search/notices

Query설명
searchRange전체 / 일주일 / 한달 / 세달
searchType제목 등
keyword검색어
page, size페이징

10. 프론트 적용 메모

상세 페이지의 "이런 제품은 어때요?" 섹션은 /products/{productId}/similar?size=3 고정 사용을 권장합니다.

UI 태그는 tags[] 순서를 그대로 렌더링하면 우선순위 정책이 자동 반영됩니다.
